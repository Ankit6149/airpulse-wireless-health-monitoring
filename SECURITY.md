# Security Policy

We take the security of AirPulse, our users, and caregivers very seriously. As a health and safety monitoring tool, protecting raw data streams and preventing vulnerabilities is of paramount importance.

---

## 🛡️ Supported Versions

We actively support security updates for the current main branch of the repository:

| Version | Supported |
| :--- | :--- |
| **0.1.x / Main** | ✅ Yes |

---

## 🔒 Reporting a Vulnerability

If you discover a security vulnerability or exploit in AirPulse (such as unauthenticated telemetry access, buffer overflows in the Go ingester, or weak database queries), please do **NOT** open a public issue. Instead, report it privately:

1. Send a private message or security note to [Ankit Bhardwaj](https://github.com/Ankit6149).
2. Detail the steps required to reproduce the vulnerability, including any mock payloads or script examples.

We will review your report within **48 hours** and coordinate a fix/patch before disclosing the vulnerability publicly. Thank you for helping keep our caregivers and seniors safe!
